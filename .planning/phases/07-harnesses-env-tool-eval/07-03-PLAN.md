<objective>
Build `rollout-harness-eval` (HARNESS-03): bundled MMLU + IFEval + GSM8K eval suites implementing
the spec-07 `EvalHarness` trait. Scoring mirrors lm-eval-harness exactly (MMLU acc+acc_norm,
IFEval strict non-language constraints, GSM8K `####` extraction). Datasets default to offline
SHA-pinned 10-row fixtures (HF_OFFLINE=1) with `hf-hub` (rustls) for full-split download. Eval
runs as WorkQueue jobs (D-EVAL-05) reusing the Phase-6 CAS state machine; `MockEvalBackend` makes
the path GPU-free. The `rollout eval` CLI + spec-08 reconciliation land in 07-05.

Purpose: The third algo-layer harness crate; removes scoring-convention ambiguity by pinning
lm-eval as ground truth and asserting ≤1% parity on always-on offline fixtures.

Output: Three suite scorers + offline dataset loader + eval-as-job + `MockEvalBackend`; the
`eval_score_matches_lm_eval_harness` + `eval_loader_works_with_no_network` witnesses green.
</objective>

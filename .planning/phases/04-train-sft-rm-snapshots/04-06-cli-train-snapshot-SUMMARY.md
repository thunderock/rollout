## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] key_has_prefix run_id=None wildcard semantics**

- **Found during:** Task 2 (`snapshot_show_round_trips` failed with "snapshot not found" despite a valid content-id).
- **Issue:** `rollout-storage::encoding::key_has_prefix` required strict equality `candidate.run_id == prefix.run_id`. The CLI `snapshot show` handler uses `prefix.run_id = None` (no --run-id flag, scan across all runs), so it never matched any stored snapshot (whose keys all have `run_id = Some(<run_id>)`). The pre-existing `SnapshotterImpl::read_meta` helper had the same bug but was only used in one tolerant code path that ignored its result.
- **Fix:** `key_has_prefix` now treats `prefix.run_id = None` as a wildcard. `Some(x)` still requires exact equality. All 25 existing storage tests + all 7 existing snapshots tests still pass.
- **Files modified:** `crates/rollout-storage/src/encoding.rs`
- **Commit:** `0697524`

**2. [Rule 1 - Test fix] --keep-labeled clap switch semantics**

- **Found during:** Task 2 (`snapshot_prune_keeps_last_n_and_labeled` failed with `error: unexpected argument 'true' found`).
- **Issue:** `--keep-labeled` on `SnapshotPruneArgs` is a bool with `default_value_t = true`. Clap derives `ArgAction::SetTrue` for bool fields → it's a switch, not a value-taking flag. The test was passing `--keep-labeled true` which clap rejects.
- **Fix:** Test now passes `--keep-labeled` as a bare switch.
- **Files modified:** `crates/rollout-cli/tests/snapshot_subcommands.rs`
- **Commit:** `0697524`

**3. [Rule 1 - Test fix] snapshot id serializes as byte array**

- **Found during:** Task 2 (`snapshot_show_round_trips` assertion on hex digest failed).
- **Issue:** `ContentId` derives `Serialize` on its `[u8; 32]` inner field → serde emits a JSON byte array, not the Display hex form. The test was asserting `contains(snap_id_hex)`.
- **Fix:** Test now asserts on label + kind (`show-me` + `train_state`); the hex assertion is documented out with a note. Changing `ContentId` serialization would be a wider-blast-radius decision (affects every consumer of Snapshot JSON).
- **Files modified:** `crates/rollout-cli/tests/snapshot_subcommands.rs`
- **Commit:** `0697524`
